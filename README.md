# Web App Inmobiliaria – QA

Proyecto de testing realizado sobre una aplicación web inmobiliaria desarrollada en un entorno IT colaborativo basado en un caso real con cliente.
Este proyecto fue desarrollado como parte de una simulación de trabajo organizada por Foo Talent Group, con roles definidos dentro del equipo y un cliente que proporcionó los requerimientos del sistema.  
El objetivo fue replicar el flujo de trabajo de un proyecto profesional incluyendo gestión de tareas, documentación funcional y testing del sistema.

## Rol
- Project Manager & Tester QA

## Alcance del trabajo
- Testing manual funcional
- Pruebas exploratorias
- Pruebas de frontend y backend
- Validación de endpoints mediante Postman
- Redacción de historias de usuario y criterios de aceptación
- Diseño y ejecución de casos de prueba

## Estrategia de Testing

El enfoque de testing se basó en validar los flujos críticos de la aplicación a partir de historias de usuario y criterios de aceptación.

El proceso incluyó:

1. Análisis de historias de usuario
2. Identificación de criterios de aceptación
3. Diseño de casos de prueba
4. Ejecución de pruebas manuales
5. Registro y seguimiento de incidencias

Las pruebas se centraron principalmente en los flujos clave del sistema inmobiliario, como la creación y validación de propiedades.

## Tipos de pruebas realizadas
- Pruebas funcionales
- Pruebas exploratorias
- Smoke testing
- Pruebas de API (Postman)

## Herramientas utilizadas
- Linear (gestión de tareas e incidencias)
- Postman (API testing)
- Excel (casos de prueba)
- Google Docs (documentación funcional)
